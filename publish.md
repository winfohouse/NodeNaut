Publishing your extension to both the Chrome Web Store and the Firefox Add-ons (Mozilla) store requires packaging your built files into  .zip  archives and submitting them through their respective developer portals.                                              
                                                                                                                                                                                                                                                                       
  To make this extremely easy for you, I have just written and added a custom packaging script to your project! You can now bundle both the Chrome and Firefox versions instantly.                                                                                     
                                                                                                                                                                                                                                                                       
### Step 1: Package Your Extensions (Automated)                                                                                                                                                                                                                      
                                                                                                                                                                                                                                                                       
  In your terminal at the root of  FlowPilot , simply run:                                                                                                                                                                                                             
                                                                                                                                                                                                                                                                       
    # Make sure your builds are up to date                                                                                                                                                                                                                             
    npm run build                                                                                                                                                                                                                                                      
    npm run build:firefox                                                                                                                                                                                                                                              
                                                                                                                                                                                                                                                                       
    # Run the new packager                                                                                                                                                                                                                                             
    npm run package                                                                                                                                                                                                                                                    
                                                                                                                                                                                                                                                                       
  This will automatically create a new  releases/  folder in your project root containing your two ready-to-upload ZIP files:                                                                                                                                          
                                                                                                                                                                                                                                                                       
  •  flowpilot-chrome-v0.2.3.zip                                                                                                                                                                                                                                       
  •  flowpilot-firefox-v0.2.3.zip                                                                                                                                                                                                                                      
                                                                                                                                                                                                                                                                       
  (Note: The version number will automatically match whatever is in your  package.json ).                                                                                                                                                                              
  ──────                                                                                                                                                                                                                                                               
  ### Step 2: Publishing to the Chrome Web Store                                                                                                                                                                                                                       
                                                                                                                                                                                                                                                                       
  1. Register as a Developer: Go to the Chrome Web Store Developer Dashboard https://chrome.google.com/webstore/devconsole/ and pay the one-time $5 registration fee (if you haven't already).                                                                         
  2. Upload your ZIP: Click "New Item" in the top right corner.                                                                                                                                                                                                        
  3. Upload the  flowpilot-chrome-v0.2.3.zip  file generated in your  releases/  folder.                                                                                                                                                                               
  4. Store Listing:                                                                                                                                                                                                                                                    
      • Enter your Title, Description (you can use our awesome new README marketing copy!), and Category (e.g., Productivity or Developer Tools).                                                                                                                      
      • Important Assets: You will need to upload:                                                                                                                                                                                                                     
          • 128x128 Icon                                                                                                                                                                                                                                               
          • 440x280 Promotional Marquee                                                                                                                                                                                                                                
          • At least one screenshot (1280x800 or 640x400)                                                                                                                                                                                                              
                                                                                                                                                                                                                                                                       
  5. Privacy Practices: Because FlowPilot runs custom scripts and accesses URLs, you will need to justify your permissions (like  activeTab ,  scripting ,  offscreen , etc.).                                                                                         
      • State clearly that data never leaves the user's machine, execution is 100% local, and the encrypted vault uses AES-256-GCM.                                                                                                                                    
  6. Submit for Review: Click "Submit for Review". Chrome usually takes 1 to 3 days to approve new extensions, especially Manifest V3 extensions with complex permissions.                                                                                             
  ──────                                                                                                                                                                                                                                                               
### Step 3: Publishing to the Firefox Add-ons Store (AMO)                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                       
  1. Register: Go to the Firefox Add-on Developer Hub https://addons.mozilla.org/en-US/developers/ and log in with a Firefox account.                                                                                                                                  
  2. Submit a New Add-on: Click "Submit a New Add-on" and accept the developer agreement.                                                                                                                                                                              
  3. Choose Distribution:                                                                                                                                                                                                                                              
      • Select "On this site" if you want it listed publicly on the Firefox store.                                                                                                                                                                                     
      • Select "On your own" if you just want Mozilla to digitally sign the  .xpi  file so you can host it yourself on your website or GitHub releases.                                                                                                                
  4. Upload your ZIP: Upload the  flowpilot-firefox-v0.2.3.zip  file from your  releases/  folder.                                                                                                                                                                     
  5. Review Warnings: Firefox might show some automated linter warnings (e.g., about using  eval()  or  new Function()  in your sandbox). Because your sandbox is properly isolated, you can usually proceed past these warnings, but a human reviewer might ask you to
  explain why you need it (you can explain it is a sandboxed IDE for user automation).                                                                                                                                                                                 
  6. Submit: Fill in your store metadata (description, screenshots, etc.) and submit. Firefox approvals are often faster than Chrome, sometimes within hours!                                                                                                          
  ──────                                                                                                                                                                                                                                                               
  ### Important Tips for Approval Success:                                                                                                                                                                                                                             
                                                                                                                                                                                                                                                                       
  • Privacy Policy: Both stores require a valid Privacy Policy URL because you request sensitive permissions. You can host a simple markdown file on GitHub Pages that states: "FlowPilot executes entirely locally. No user data, automation scripts, or browsing     
  history is collected, transmitted, or sold by the developer."                                                                                                                                                                                                        
  • Source Code (Firefox): If your code is minified (Vite does this by default), Mozilla may require you to upload the original source code. You can just zip up your whole project (excluding  node_modules  and  dist ) and upload it in the "Source Code" section   
  when submitting to Firefox.
